它以极简的语法规则，赋予普通文本灵动的结构与优雅的格式。

## 一、基础文本格式化

基础文本格式化是Markdown的基石，它让文字摆脱平淡，拥有强调、区分和情感表达的能力。掌握这些简单的语法，便能轻松提升文本的可读性与表现力。

### 1.1 标题

标题是文档结构的核心，Markdown提供了六级标题语法，通过在文字前添加“#”符号来表示，“#”的数量越多，标题级别越低，字体越小。

- `# 一级标题：对应HTML中的<h1>标签，是文档中最高级别的标题，通常用于文章大标题，字体最大、最醒目。`

- `## 二级标题：对应<h2>标签，常用于章节标题，字体大小仅次于一级标题。`

- `### 三级标题：对应<h3>标签，适用于小节标题，层次更细致。`

- `#### 四级标题：对应<h4>标签，用于更细分的内容板块。`

- `##### 五级标题：对应<h5>标签，字体较小，可用于次要的子标题。`

- `###### 六级标题：对应<h6>标签，是最低级别的标题，字体最小，适用于细节说明性标题。`

需要注意的是，“#”与标题文字之间必须保留一个空格，否则将被视为普通文本。此外，一级标题和二级标题还可以在文字下方添加“=”和“-”来表示，例如：

```Plain Text

一级标题示例
=
二级标题示例
-
```

### 1.2 段落与换行

在Markdown中，段落的划分非常简单，只需在两段文字之间保留一个或多个空行即可。如果希望在同一段落内实现换行（软换行），则需要在换行处添加两个或多个空格，然后按下回车键。例如：

```Plain Text

这是第一段文字，通过空行与第二段分隔。

这是第二段文字，这里需要换行  
这是换行后的内容（注意换行处有两个空格）
```

### 1.3 强调

为了突出文本中的重点内容，Markdown提供了斜体、粗体和粗斜体三种强调方式，分别通过不同数量的“*”或“_”符号包裹文字实现。

- **斜体**：用一个“*”或“_”包裹文字，例如`*斜体文本*`或`_斜体文本_`，渲染后效果为*斜体文本*。

- **粗体**：用两个“*”或“_”包裹文字，例如`**粗体文本**`或`__粗体文本__`，渲染后效果为**粗体文本**。

- **粗斜体**：用三个“*”或“_”包裹文字，例如`***粗斜体文本***`或`___粗斜体文本___`，渲染后效果为***粗斜体文本***。

需要注意的是，“*”和“_”必须成对使用，且与文字之间不能有空格，否则强调效果不生效。

### 1.4 引用

当需要引用他人的话语、文献内容或突出某段重要观点时，可以使用引用语法。在引用文本前添加“>”符号，并在符号后保留一个空格即可。引用还支持嵌套，只需在嵌套的引用文本前添加多个“>”符号。

```Plain Text

> 这是一级引用，用于直接引用一段文字。
> 
> > 这是嵌套的二级引用，可以用于引用中的引用，层次分明。
> 
> 引用结束后，只需空一行即可回到普通文本格式。
```

渲染效果如下：

> 这是一级引用，用于直接引用一段文字。
> 
> 这是嵌套的二级引用，可以用于引用中的引用，层次分明。
> 
> 

引用结束后，只需空一行即可回到普通文本格式。

## 二、列表

列表是组织内容的重要工具，Markdown支持有序列表和无序列表，以及列表的嵌套，能够清晰地呈现步骤、要点等结构化信息。

### 2.1 无序列表

无序列表用于呈现没有先后顺序的要点，可使用“*”“+”或“-”作为列表项的标记，标记后需保留一个空格再填写列表内容。三种标记的效果相同，可根据个人习惯选择。

```Plain Text

* 无序列表项1（使用*标记）
+ 无序列表项2（使用+标记）
- 无序列表项3（使用-标记）
```

渲染效果如下：

- 无序列表项1（使用*标记）

- 无序列表项2（使用+标记）

- 无序列表项3（使用-标记）

### 2.2 有序列表

有序列表用于呈现具有先后顺序的内容，如步骤、流程等。在列表项前添加数字和英文句点“.”，句点后保留一个空格再填写内容。需要注意的是，数字的实际值并不影响列表的顺序，渲染时会自动按照连续的数字排列。

```Plain Text

1. 有序列表第一步
2. 有序列表第二步
3. 有序列表第三步
```

渲染效果如下：

1. 有序列表第一步

2. 有序列表第二步

3. 有序列表第三步

### 2.3 列表嵌套

当列表项需要进一步细分时，可以使用列表嵌套。在嵌套的列表项前添加四个空格或一个制表符（Tab），即可实现嵌套效果，支持有序列表与无序列表之间的交叉嵌套。

```Plain Text

1. 主列表项1
   * 嵌套无序列表项1
   * 嵌套无序列表项2
2. 主列表项2
   1. 嵌套有序列表项1
   2. 嵌套有序列表项2
```

渲染效果如下：

1. 主列表项1
        嵌套无序列表项1

2. 嵌套无序列表项2

3. 主列表项2
        嵌套有序列表项1

4. 嵌套有序列表项2

### 2.4 任务列表：待办事项的清晰追踪

任务列表（待办列表）是一种特殊的列表形式，用于追踪待办事项的完成状态。在列表项前添加“- [ ]”表示未完成任务，添加“- [x]”（x不区分大小写）表示已完成任务，中括号内与括号之间需保留空格。

```Plain Text

- [x] 完成Markdown基础语法学习
- [ ] 学习Markdown进阶语法
- [ ] 实践撰写一篇Markdown文档
```

渲染效果如下：

- [x] ~~完成Markdown基础语法学习~~

- [ ] 学习Markdown进阶语法

- [ ] 实践撰写一篇Markdown文档

## 三、代码块

对于程序员和技术文档撰写者而言，代码块是不可或缺的功能。Markdown提供了行内代码和块级代码两种展示方式，支持语法高亮，让代码呈现更加清晰易读。

### 3.1 行内代码：嵌入文本的代码片段

行内代码用于在普通文本中嵌入简短的代码片段，只需用反引号“`”将代码包裹起来即可。如果代码片段中本身包含反引号，可以使用两个反引号“``”包裹。

示例：`在Python中，使用`print("Hello, Markdown!")`可以输出问候语。`，渲染效果为：在Python中，使用`print("Hello, Markdown!")`可以输出问候语。

### 3.2 块级代码

块级代码用于展示完整的代码片段，有两种表示方式：

- **缩进式**：在代码块的每一行前添加四个空格或一个制表符（Tab）。这种方式简单直接，但对于多行代码的缩进调整较为繁琐。

- **围栏式**：使用三个或更多反引号“```”包裹代码块，在开头的反引号后可以指定编程语言，实现语法高亮。这种方式是目前最常用的代码块表示方法，支持多种编程语言的语法高亮。

```Plain Text

// 缩进式代码块（每行前加四个空格）
function add(a, b) {
    return a + b;
}
```

```Plain Text

```javascript
// 围栏式代码块（指定JavaScript语言，支持语法高亮）
function add(a, b) {
    return a + b;
}
console.log(add(2, 3)); // 输出5
```
```

围栏式代码块渲染效果如下（带有语法高亮）：

```javascript

// 围栏式代码块（指定JavaScript语言，支持语法高亮）
function add(a, b) {
    return a + b;
}
console.log(add(2, 3)); // 输出5
}
```

## 四、链接与图片

链接让文档与外部世界相连，图片则为文档增添视觉色彩。Markdown提供了简洁的语法来插入链接和图片，让文档更加丰富多元。

### 4.1 链接

Markdown的链接语法分为内联链接、参考链接和锚点链接三种类型。

#### 4.1.1 内联链接

内联链接是最常用的链接形式，语法格式为`[链接文本](链接地址 "可选的标题")`。其中，“链接文本”是显示在文档中的可点击文字，“链接地址”是跳转的目标URL，“可选的标题”在鼠标悬停链接时显示，用双引号或单引号包裹。

示例：`[Markdown官方文档](https://www.markdownguide.org "Markdown Guide")`，渲染效果为：[Markdown官方文档](https://www.markdownguide.org "Markdown Guide")。

#### 4.1.2 参考链接

参考链接适用于同一链接在文档中多次使用的场景，通过定义链接标签来复用链接地址，使文档代码更加简洁。语法格式分为两部分：链接的引用部分和链接的定义部分。

- **引用部分**：`[链接文本][链接标签]`，“链接标签”可以是字母、数字、空格或标点符号（不区分大小写）。

- **定义部分**：`[链接标签]: 链接地址 "可选的标题"`，定义部分可以放在文档的任何位置，通常放在文档末尾。

```Plain Text

我推荐阅读[Markdown官方文档][md-guide]，也可以参考[GitHub Flavored Markdown][gfm]文档。

[md-guide]: https://www.markdownguide.org "Markdown Guide"
[gfm]: https://github.github.com/gfm/ "GitHub Flavored Markdown"
```

渲染效果与内联链接相同：我推荐阅读[Markdown官方文档][md-guide]，也可以参考[GitHub Flavored Markdown][gfm]文档。

[md-guide]: https://www.markdownguide.org "Markdown Guide"
[gfm]: https://github.github.com/gfm/ "GitHub Flavored Markdown"

#### 4.1.3 锚点链接

锚点链接用于在同一文档内跳转到指定的锚点位置，实现文档内部的快速导航。语法格式为`[链接文本](#锚点名称)`，其中“锚点名称”通常是文档中标题的小写形式，单词之间用连字符“-”连接（不同解析器对锚点的处理可能略有差异，需根据实际情况调整）。

示例：`[跳转到“基础文本格式化”部分](#一基础文本格式化：为文字注入灵魂)`，点击后将跳转到本文中“一、基础文本格式化：为文字注入灵魂”的标题位置。

### 4.2 图片

图片的语法与链接类似，只是在链接语法前多了一个感叹号“!”。语法格式为`![图片 alt 文本](图片地址 "可选的图片标题")`。

- **图片 alt 文本**：当图片无法加载时显示的替代文本，对 accessibility（可访问性）非常重要。

- **图片地址**：可以是本地图片的路径（相对路径或绝对路径），也可以是网络图片的URL。

- **可选的图片标题**：与链接的标题类似，在鼠标悬停图片时显示。

示例：`![Markdown Logo](https://markdown-here.com/img/icon256.png "Markdown Logo")`，渲染效果为：

与链接类似，图片也支持参考式语法，格式为`![图片 alt 文本][图片标签]`，然后在文档其他位置定义`[图片标签]: 图片地址 "可选的图片标题"`。

## 五、表格

表格是呈现结构化数据的理想方式，Markdown通过简单的符号组合，便能创建清晰美观的表格。表格由表头、分隔线和表体三部分组成。

### 5.1 基本表格语法

```Plain Text

| 表头1 | 表头2 | 表头3 |
| --- | --- | --- |
| 单元格1 | 单元格2 | 单元格3 |
| 单元格4 | 单元格5 | 单元格6 |
```

其中，“|”用于分隔列，“---”用于表示表头与表体之间的分隔线。分隔线的长度不影响表格的渲染效果，但为了美观，通常与列的宽度保持一致。

渲染效果如下：

|表头1|表头2|表头3|
|---|---|---|
|单元格1|单元格2|单元格3|
|单元格4|单元格5|单元格6|
### 5.2 表格对齐方式

Markdown支持表格列的对齐方式设置，通过在分隔线中添加冒号“:”来实现，冒号的位置决定了对齐方向：

- `:---`：左对齐（默认对齐方式）

- `:---:`：居中对齐

- `---:`：右对齐

```Plain Text

| 左对齐 | 居中对齐 | 右对齐 |
| :--- | :---: | ---: |
| 内容1 | 内容2 | 内容3 |
| 长文本内容示例 | 居中文本 | 数值100 |
```

渲染效果如下：

|左对齐|居中对齐|右对齐|
|---|---|---|
|内容1|内容2|内容3|
|长文本内容示例|居中文本|数值100|
## 六、扩展语法：解锁更多高级功能

除了上述基础语法外，不同的Markdown解析器还支持一些扩展语法（如GitHub Flavored Markdown，简称GFM），这些语法为Markdown增添了更多实用功能，如脚注、定义列表、删除线等。

### 6.1 脚注

脚注用于为文档中的内容添加补充说明，不影响正文的阅读流畅性。语法格式为：在需要添加脚注的位置使用`[^脚注标识]`，然后在文档末尾定义脚注内容`[^脚注标识]: 脚注具体内容`。脚注标识可以是字母、数字或符号，无需连续。

```Plain Text

Markdown是一种轻量级标记语言[^1]，由John Gruber和Aaron Swartz共同设计[^2]。

[^1]: 轻量级标记语言是一类用简单标记描述文档结构和格式的语言，相比HTML更加简洁易读。
[^2]: John Gruber是一位技术作家和博主，Aaron Swartz是一位程序员和互联网活动家。
```

渲染效果中，脚注标识会显示为上标链接，点击可跳转到文档末尾的脚注内容处。

### 6.2 定义列表：术语与解释的配对

定义列表用于呈现术语及其解释，语法格式为：术语独占一行，解释部分在术语下方，以冒号“:”开头，并缩进四个空格或一个制表符。

```Plain Text

Markdown
: 一种轻量级标记语言，用于格式化文本。

GFM
: GitHub Flavored Markdown的缩写，是GitHub对标准Markdown的扩展。
```

渲染效果如下：

Markdown
: 一种轻量级标记语言，用于格式化文本。

GFM
: GitHub Flavored Markdown的缩写，是GitHub对标准Markdown的扩展。

### 6.3 删除线

删除线用于标记已过时、错误或需要删除的内容，语法格式为用两个波浪号“~~”包裹文本。

示例：`~~这是需要删除的内容~~`，渲染效果为：~~这是需要删除的内容~~。

### 6.4 高亮

部分Markdown解析器（如GFM）支持高亮语法，用两个等号“==”包裹文本即可实现高亮效果。

示例：`这是==需要高亮的重点内容==`，渲染效果为：这是==需要高亮的重点内容==（具体效果取决于解析器是否支持）。

### 6.5 数学公式

对于科技类文档，数学公式是重要的组成部分。Markdown通常通过LaTeX语法来支持数学公式，分为行内公式和块级公式。

- **行内公式**：用单个美元符号“$”包裹LaTeX公式，例如`$E=mc^2$`，渲染效果为：$E=mc^2$。

- **块级公式**：用两个美元符号“$$”包裹LaTeX公式，公式将独占一行并居中显示，例如：

```Plain Text

$$
\int_{a}^{b} f(x) \, dx = F(b) - F(a)
$$
```

渲染效果为：$$\int_{a}^{b} f(x) \, dx = F(b) - F(a)$$（需解析器支持LaTeX公式渲染）。

## 七、实用技巧与注意事项

掌握了Markdown的语法规则后，了解一些实用技巧和注意事项，能让你的文档排版更加规范、美观。

### 7.1 水平线

在文档中添加水平线可以清晰地分割不同的内容板块，语法格式为在单独一行中输入三个或更多的“*”“-”或“_”，符号之间可以有空格。

```Plain Text

***
---
___
```

三种方式渲染效果相同，均为一条水平线：

---

### 7.2 转义字符

当需要在文档中显示Markdown的特殊符号（如“#”“*”“[”“]”等）时，需要使用反斜杠“\”进行转义，让解析器将其识别为普通字符。

示例：`\# 这不是标题`，渲染效果为：# 这不是标题；`\*这不是斜体\*`，渲染效果为：*这不是斜体*。

### 7.3 空格与空行

Markdown对空格和空行较为敏感，合理使用能提升文档的排版效果：

- 标题语法中，“#”与标题文字之间必须保留一个空格。

- 列表项标记、引用标记与内容之间必须保留一个空格。

- 段落之间需保留一个或多个空行才能正确划分段落。

- 同一段落内换行需在换行处添加两个或多个空格。

### 7.4 解析器差异

不同的Markdown解析器对语法的支持可能存在差异，尤其是扩展语法部分。例如，GFM支持删除线、表格等扩展语法，而一些古老的解析器可能仅支持基础语法。在撰写文档时，需根据目标平台（如GitHub、CSDN、简书等）的解析器特性，选择合适的语法，以确保文档的正确渲染。

## 八、总结

Markdown以其极简的语法规则，实现了“易写易读”的核心目标。它摒弃了复杂的排版操作，让使用者能够专注于内容创作本身，同时又能轻松生成结构清晰、格式优美的文档。从基础的文本格式化到进阶的多媒体嵌入，从简单的列表到复杂的表格和数学公式，Markdown用简洁的符号构建了一个强大的排版体系。
